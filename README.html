01 What Is This?

In Atomic Force Microscopy (AFM), the measured surface is never the true surface. The physical tip has a finite size and shape — it cannot fit into features narrower than itself, and it adds its own geometry to every feature it scans over. This effect is called a tip–sample convolution artifact.

This simulator lets you explore those artifacts interactively. You choose a tip geometry and a surface profile, watch the scan animate in real time, and immediately see how much the measured surface deviates from the truth — quantified as max artifact and RMS error.

Two simulators
2D Simulator — Cross-sectional scan (one line profile). Fast, detailed, shows the tip moving in real time. Best for understanding the physics.

3D Simulator — Full raster scan over an 80×80 nm grid. Builds the measured surface row by row. Shows 3D tip geometry and exports mesh files.
02 Physics & Formula

AFM contact mode imaging is modelled as a Minkowski dilation (or erosion) of the surface by the tip profile. The correct formula is:

Core AFM Minkowski Formula
M[i] = maxj { S[i+j] − T[j] }
M[i]
Measured (artifact-distorted) surface height at scan position i
S[i+j]
True surface height at offset j from position i
T[j]
Tip profile height above apex at lateral offset j (always ≥ 0)
Why subtract T, not add?
On a perfectly flat surface, the measured height should equal the true height. With the correct formula: M[i] = maxj{BASE − T[j]} = BASE − min(T) = BASE − 0 = BASE ✓
If you add T, a flat surface gives M = BASE + T_max — completely wrong. The subtraction is the physically correct Minkowski dilation direction.

For the Concave Sphere tip only, the operation switches to Minkowski erosion:

Erosion (Concave Sphere only)
M[i] = minj { S[i+j] + T[j] }

The Invert toggles in the sidebar let you flip between dilation and erosion manually for any tip, enabling exploration of deconvolution effects.

03 Tip Geometries — 8 Types

Each tip is defined by its profile function T(r), where r is the lateral (horizontal) distance from the apex. T(r) is always zero at the apex (r = 0) and increases as you move away. Blocked positions (tip body cannot reach) are set to infinity.

▼Cone
T(r) = r / tan(α)
Pure conical tip. Half-angle α from the tip axis (vertical). Small α = very sharp needle. Large α = blunt wide cone. Slider range: 3°–80°, default 15°.
↳ Broadens all features. Peaks widen by r·tan(α) on each side.
●Spherical
T(r) = R − √(R²−r²) for r < R
Hemispherical apex. Radius R in nm. Typical commercial AFM probes have R = 5–30 nm. The sphere only interacts within r < R.
↳ Smooth rounding of sharp features. Broadening radius ≈ R.
⌓Hyperboloid
T(r) = √(r² + a²) − a
Hyperboloidal tip with curvature parameter a. Intermediate between cone and sphere. Sharp at apex, curves smoothly outward.
↳ Intermediate broadening. Sharper than sphere, softer than cone.
▬Flat Punch
T(r) = 0 for r ≤ R, ∞ for r > R
Flat circular face of radius R. Cannot resolve features narrower than 2R (the full punch diameter). Sharp edge creates characteristic plateaus.
↳ Flattens peaks. Features < 2R appear wider and flattened.
⌣Concave Sphere
T(r) = R − √(R²−r²) [EROSION mode]
A spherical bowl (concave). The rim contacts the surface, not the centre. Uses erosion operation — measured surface is BELOW the true surface. Purple in the simulator.
↳ Narrowing artifact. Features appear thinner (erosion).
↗Asymmetric
T(dc) = |dc|/tan(α) + dc·tan(φ)
Tilted cone. α = half-angle, φ = tilt angle. The signed offset dc makes one side steeper, one shallower. Creates directional artifacts — features appear shifted.
↳ Lateral shift. Features displaced in the tilt direction.
⌤Double Tip
T(dc) = min(|dc−g|, |dc+g|) / tan(α)
Two identical apexes separated by gap = 2g. T = 0 at both dc = ±g (both apex positions). Centre between tips is raised by g/tan(α).
↳ Ghost image. Every surface feature appears TWICE at ±gap offset.
◓Cone + Sphere (U-Shape)
Arc: T = R−√(R²−r²) · Side: T = yt+(r−w)·w/sv
Exact SolidWorks U-shape geometry. Circular arc of radius R at apex, C1-continuous join to straight slanted sides at tangent half-width w. Top width = 2w. Centre marked in purple.
↳ Combined rounding + broadening. Arc-dominated at narrow features.
04 Surface Profiles — 10 Types

All surfaces are defined in the range 0–240 nm (horizontal) and sit on a base height of 25 nm. Amplitudes are set so features stay in the 10–70 nm range, well within the 100 nm display window.

Best combinations to try
Sharp Cone + Spike — Maximum broadening artifact, very obvious.
Flat Punch + DNA Ridges — Ridges narrower than 2R disappear entirely.
Double Tip + Nanoparticle — Classic ghost image, two bumps from one.
Sphere + Lattice — Resolution limit visible as lattice fades with period.
Concave Sphere + Hemisphere — Erosion makes the dome appear narrower.
Asymmetric + Step Edge — Step looks higher on one side than the other.
05 2D Simulator — Step by Step
1
Select a Tip Geometry
Click any tip button in the left sidebar. The tip shape updates instantly in the preview thumbnail below the buttons. Adjust the sliders that appear (e.g. radius R, half-angle α) to customise the tip size and shape. The tip shape redraws live as you drag.
2
Select a Surface Profile
Click a surface button. The true surface (teal line) updates immediately in the main canvas. Adjust amplitude, period, depth etc. with the surface sliders. The surface always sits above the 25 nm baseline.
3
Press PLAY
The tip (shown as a coloured silhouette) scans from left to right. The orange dashed line shows the AFM-measured surface building up in real time. The white dot at the tip apex glows green when touching the true surface, orange when floating above it (artifact gap).
4
Read the Statistics
Max Artifact — largest single-point deviation (nm).
RMS Error — root-mean-square error across the full scan (nm).
These update live during the scan. A flat surface with any tip should give 0 nm artifact.
5
Experiment with Invert Toggles
The six toggle switches change how the Minkowski operation is applied. Erosion mode swaps dilation for erosion — measured surface drops below true. Inv Tip negates the tip profile. See Section 07 for all toggle details.
6
Try the Scan Modes
ONCE — single left→right pass then stops.
LOOP — continuous left→right, resets at end.
BOUNCE — scans left→right then right→left, like real AFM bidirectional scanning.
Three canvas panels
Main canvas (top) — True surface (teal) and measured surface (orange dashed) with the scanning tip silhouette. A dashed vertical line shows where the tip currently is.

Error bar (middle) — Per-pixel artifact magnitude as a bar chart. Tall bars = large artifacts at that x position.

FD Curve (bottom) — Force–distance approach curve for the tip at the current scan position.
06 3D Simulator — Step by Step
1
Select Tip and Surface
Same as 2D — click buttons in the left sidebar. The 3D tip geometry updates immediately in the viewport. Adjust sliders for tip and surface parameters. The surface mesh rebuilds instantly.
2
Press PLAY — Watch the Raster Scan
The tip (3D mesh) scans across the surface row by row, left to right, front to back. Each completed row appears in orange as the measured surface builds up. The scan covers an 80×80 grid over a 200×200 nm area.
3
Orbit the View
Drag — orbit around the scene.
Scroll — zoom in and out.
ISO / TOP / FRONT buttons — snap to preset views.
Use TOP view to compare true and measured surface morphology from above.
4
Adjust Vertical Exaggeration
The V. Exaggeration slider (0.5×–10×) scales the Z axis of the surface mesh. Increase it to make shallow features visible. The true scan physics are not affected — it's purely a display scaling for visual clarity.
5
Export Meshes
Two export buttons below the view controls:
⬇ TIP .OBJ — downloads the current tip mesh as a Wavefront .obj file, named with the tip type and parameters (e.g. tip_sphere_cone_R30_angle15.obj).
⬇ SURF .OBJ — downloads the current surface mesh.
07 All Controls Explained

The six INVERT / TRANSFORM toggles modify the scan operation:

Inv Z
Flips all surface heights. Peaks become valleys and vice versa. Useful for studying how tip–sample interaction changes on inverted topography.
Mirror X
Flips the surface left–right (horizontal mirror). Useful with asymmetric tips to see directional artifact reversal.
Mirror Y
(3D only) Flips the surface front–back. Combined with Mirror X gives a 180° rotation of the scan pattern.
Inv Tip
Negates the tip profile T(r) → −T(r). Combined with Erosion it effectively deconvolves: estimates the true surface from a measured one.
Rev Scan
Reverses the scan direction (right→left instead of left→right). Shows how directional artifacts depend on scan orientation — important for asymmetric tips.
Erosion
Switches from Minkowski dilation (M = max{S−T}) to erosion (M = min{S+T}). Measured surface falls BELOW true surface. The Concave Sphere tip always uses erosion automatically.
ControlEffect on physicsNotes
Speed slider Scan pixels per animation frame Range 1–12. Higher = faster scan but coarser animation.
Noise slider Adds ±noise_amp Gaussian noise to surface 0 nm = clean. 3–5 nm = realistic lab conditions.
V. Exaggeration (3D) Scales Z display only — no physics change 1× = true proportions. 5–10× useful for shallow surfaces.
🎲 Randomize Picks random tip + surface + parameters Good for exploring unexpected combinations quickly.
↺ Reset Resets scan to start position Does not change tip or surface selection.

08 Exporting Meshes (.OBJ)

The 3D simulator can export any tip or surface mesh as a standard Wavefront .OBJ file, which can be opened in Blender, MeshLab, SolidWorks, FreeCAD, 3D printing slicers, and Python via trimesh or open3d.

How to export
1. Open the 3D Simulator (afm-3d.html)
2. Select the tip and surface you want
3. Adjust sliders to the exact geometry you need
4. Click ⬇ TIP .OBJ to download the tip mesh
5. Click ⬇ SURF .OBJ to download the current surface mesh

The tip filename includes the tip type and parameter values:
tip_sphere_cone_R30_angle15.obj
surface_sine.obj

The .OBJ file contains:

SectionContent
v x y zVertex positions in 3D space (nm units)
vn x y zVertex normals (computed automatically for smooth shading)
f a//a b//b c//cTriangular faces with vertex/normal indices (1-based)
Loading in Python
import trimesh
mesh = trimesh.load('tip_sphere_cone_R30_angle15.obj')
print(mesh.vertices.shape, mesh.faces.shape)
09 Dataset Generator

A Python script (afm_dataset_generator.py) generates large batches of AFM simulation images using the same physics as the simulator. Each sample produces 5 images and a JSON parameter file.

Output per sample
true_surface_16bit.png — Ground-truth height map (16-bit lossless)
true_surface_rgb.png — Teal-cyan false colour
measured_afm_16bit.png — AFM-measured height map (16-bit lossless)
measured_afm_rgb.png — Orange false colour
error_map_rgb.png — Artifact error (blue → green → red)
params.json — All parameters + statistics (err_rms, err_max, …)

Install dependencies:

pip install numpy pillow tqdm scipy

Generate all 80 combinations (8 tips × 10 surfaces), 100 samples each at 256×256:

Shell loop — organised subfolders
for TIP in cone sphere hyperboloid flat inv_sphere asymmetric double_tip sphere_cone; do
  for SURF in sine hills hemisphere pit trench rough pyramid dna chirp lattice; do
    python afm_dataset_generator.py \
      --n_samples 100 --grid_size 256 \
      --tip $TIP --surf $SURF \
      --out_dir dataset/${TIP}__${SURF}
  done
done

Or all at once with joblib parallelism:

Parallel — all cores
python afm_dataset_generator.py --parallel --n_per_combo 100 --grid_size 256

10 Quick Reference
TermFormula / ValueDescription
Dilation M[i] = max_j { S[i+j] − T[j] } Standard AFM contact mode. Measured ≥ True.
Erosion M[i] = min_j { S[i+j] + T[j] } Concave tip or deconvolution. Measured ≤ True.
Base height 25 nm All surfaces sit at this height at their lowest.
Display range 0–100 nm Y axis of the 2D and 3D display.
Scan width 240 nm (2D) / 200 nm (3D) Total horizontal extent of the simulated scan.
Grid size 700 px (2D) / 80×80 (3D) Number of scan positions.
Kernel radius 30 nm Maximum lateral extent of tip interaction.
T_clip 75 nm Kernel values above this are treated as blocked (∞).
Common misconceptions
❌ "The measured surface is always higher than the true surface."
Only for dilation mode. Erosion (concave sphere, or Erosion toggle) gives measured < true.

❌ "A sharper tip always gives less artifact."
True for broadening. But a very sharp cone has a large T_MAX, causing severe broadening of tall features. A blunt sphere may outperform a sharp cone on smooth surfaces.

❌ "The flat punch has zero artifact."
On a flat surface, yes. On any feature narrower than 2R, the flat punch cannot resolve it — it measures a widened, flattened version.
